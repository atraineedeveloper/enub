## 11. Shared worker route gate

**Historical note (tasks 11.1–11.4 below, as originally written, described a page-level/render-prop gate design that no longer exists — see design.md §16 and §15 Item 2 for why, and the corrected description that follows).** The original 11.1–11.4 evaluated and shipped `WorkerRouteGate` as a component wrapped *around each page's content*, using a render-prop (`children: (workerId: number) => ReactNode`) to hand the session-derived `workerId` down to that page, with the redirect condition written as `!(role === "worker" && workerId != null)`. A later audit found two problems with that shape: (1) `WorkerAppLayout` — and therefore `Header`/`WorkerNav` — could mount before the gate had resolved, since the gate lived *inside* the layout, not above it; (2) `workerId != null` admits `0`/negative/non-integer/`NaN`/`Infinity` as "valid," which is weaker than the finite-positive-integer check used elsewhere. Both were fixed; 11.1–11.4 as originally worded are **superseded** by the corrected tasks below, not current requirements. They are struck through rather than deleted so the evaluation history (why a shared gate was chosen at all, over three duplicated copies) remains visible.

### Corrected architecture (current, approved — design.md §16/§17, tasks §15 Item 2)

- [x] 11.1′ `WorkerRouteGate` (`src/ui/WorkerRouteGate.tsx`) is a **layout route** wrapping the entire worker `<Route>` branch, above `WorkerAppLayout` — not a per-page wrapper: `<Route element={<WorkerRouteGate />}><Route element={<WorkerAppLayout />}>...`.
